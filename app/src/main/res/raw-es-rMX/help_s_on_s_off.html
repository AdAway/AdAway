Dispositivos HTC

AdAway no funciona cuando utilizas dispositivo con S-ON. Esta "característica" existe en varios dispositivos HTC y evita que AdAway escriba el archivo hosts.

S-ON/S-OFF?

S-OFF significa que la porción NAND del dispositivo está desbloqueada y puede ser escrito en ella. La configuración predeterminada para dispositivos HTC es S-ON, lo que significa que no se puede acceder a ciertas áreas del sistema ni tampoco se puede garantizar el root permanente. Además, la comprobación de la firma para las imágenes de firmware también está garantizada por el indicador S-ON.

¿Tengo S-ON or S-OFF?

Reinicia al Boot Menu en tu dispositivo manteniendo presionando el botón de bajar volumen mientras presionas el botón de encendido y el texto en la parte superior mostrará el estado del indicador como S-OFF o S-ON. Un dispositivo completamente rooteado se mostrará como S-OFF.

Puedes encontrar más información en www.addictivetips.com. Métodos S-OFF adicionales con Unrevoked (en el link) pueden interesarte, es decir: Revolutionary, Revone, Firewater, RumRunner, Moonshine, SunShine...

Solución

Pre requisito: Debes instalar y ejecutar el SDK de Android con el ADB shell en tu PC.

  1. Accede al Boot Menu de tu dispositivo manteniendo el botón de bajar volumen mientras presionas el botón de apagado.
  2. Usa la tecla de de bajar volumen para seleccionar recovery.
  3. En el clockwork recovery baja con la tecla de volumen hasta "partitions menu" y pulsa el botón del track pad para seleccionar
  4. Selecciona mount/system, mount /sdcard y mount/data.
  5. Conecta por cable USB y abre una línea de comando en tu PC.
  6. Entra al adb shell y escribe ln -s /data/hosts /system/etc/hosts (Esto crea un link simbólico que permite a AdAway editar el archivo hosts almacenado en /data mientras le permite al SO usar el archivo como si estuviera almacenado en /system.)
  7. Reinicia tu dispositivo y selecciona el Archivo Hosts para /data/hosts en las preferencias de AdAway.
  8. AdAway debería funcionar ahora.
