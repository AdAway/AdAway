Dispositivos HTC

AdAway não funciona quando você está usando um dispositivo com então chamado S-ON. Este 'recurso' existe em muitos dispositivos HTC e evita AdAway de escrever o arquivo hosts.

S-ON/S-OFF?

S-OFF significa que a partição NAND do dispositivo está desbloqueada e pode ser escrita. A definição padrão para dispositivos HTC é S-ON, que significa que você não pode acessar certas áreas do sistema nem garantir um root permanente. Além disto, a verificação de assinatura para imagens de firmware é também assegurada pela bandeira S-ON.

Tenho S-ON ou S-OFF?

Inicialize dentro do Menu de Inicialização no seu dispositivo pressionando para baixo botão de diminuir volume enquanto pressiona o botão de energia e o texto no topo mostrará o status da bandeira como qualquer um S-OFF ou S-ON. Um root completo geralmente significa S-OFF.

Mais informação pode ser encontrada em www.addictivetips.com. Métodos S-OFF adicionais além do Irrevogável (no link) podem interessar você, nomeadamente: Revolutionary, Revone, Firewater, RumRunner, Moonshine, SunShine...

Solução Alternativa

Pré-Requisito: Você tem que instalar um Android SDK funcional com ADB shell no seu PC.

  1. Inicialize dentro do Menu de Inicialização no seu dispositivo pressionando botão de diminuir volume enquanto pressiona o botão de energia.
  2. Use diminuir volume para selecionar a recuperação.
  3. No mecanismo de recuperação use diminuir volume no "menu de partições" e use o trackpad para selecionar.
  4. Selecione mount /system, mount /sdcard e mount /data.
  5. Conecte seu cabo USB e abra uma linha de comando no seu computador.
  6. Entre no ADB Shell e digite ln -s /data/hosts /system/etc/hosts (Isso cria um link simbólico que permite que o AdAway edite o arquivo hosts que é guardado em /datapermitindo que o OS use o arquivo como se ele estivesse guardado em /system.)
  7. Reinicie seu dispositivo e defina Arquivo de hosts alvo para /data/hosts nas preferências do AdAway.
  8. AdAway deve funcionar agora.
