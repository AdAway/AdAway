在 Android 9+ 中複製 hosts 檔案失敗

最新的 Android 版本中使用了唯讀的 /system分割區
。 如果你使用 Magisk 當作 root  的解決方案，請確定你已啟用 systemless hosts 模組並重啟。

分割區上無足夠的空間

請在設定中將目標 hosts 檔案的位置變更為 /data/data/hosts（或 /data/hosts），並且再次套用 AdAway。

無法在 Android 4.4+ 版以上運作

確保從 Chrome 設定中停用精簡模式（以前稱為 data saver）。這個特性允許 Chrome 使用私有 DNS 解決方案並繞過 AdAway。

在某些應用程式中仍然看的到廣告！

有些hostnames可能不在hosts 來源的hosts 檔案之中或廣告圖像被包括在應用程式之中，以至不用上網也可顯示廣告。

你可以AdAway(Menu->記錄 DNS 請求) 記錄DNS請求去找出需要阻擋的hostnames。

開啟記錄檔後，長按該主機名稱就能將可疑的主機加入至個人黑名單中，如果某台主機確認為廣告主機後，你可以透過下面的論壇連結回報給我們。 hosts-file.net的 Hosts Inbox。

已存於快取中的廣告

某些軟體在下載時，會同時將廣告儲存在快取檔中，因此在這些軟體仍然會有廣告出現。你可以透過 Android 應用程式清單中的刪除快取檔案來解決此問題。

開啟廣告阻擋後，某些應用程式無法執行了！

有些應用程式需要連線到被 AdAway 阻擋的主機名稱，或廣告主機被阻擋導致程式拒絕執行。請參考 https://github.com/AdAway/AdAway/wiki/ProblematicApps 中記載有問題的應用程式清單。或是找出需要連線的主機名稱後，將它加入到你的名單中的白名單，並回報至 AdAway BUG追蹤頁面。

退出功能在瀏覽網頁時是無效的。

可在 AdAway 的設定中啟用本地網頁伺服器作為解決辦法。
