在 Android 9+ 上複製 hosts 檔案失敗

新版的 Android 使用唯讀的 /system 分割區
。 如果您使用 Magisk 作為 root 解決方案，請確保啟用無系統 hosts 模組，然後重新啟動手機。

分割區空間不足

嘗試在偏好設定中將目標 hosts 檔案更改為 /data/data/hosts（或 /data/hosts），然後再次應用 AdAway。

Chrome 仍然顯示廣告

請在 Chrome 設定中停用lite 模式（以前稱為data saver）。此功能使 Chrome 可以使用私有 DNS 解決方案，從而繞過 AdAway。

應用程式 XYZ 中未阻擋廣告！

有些hostnames可能不在hosts 來源的hosts 檔案之中或廣告圖像被包含在應用程式之中，以至不用網絡連線也可顯示廣告。

你可以中找出AdAway(Menu->DNS 請求記錄) DNS請求記錄去找出需要阻擋的hostnames。

開啟記錄檔後，長按該主機名稱就能將可疑的hostnames加入至個人黑名單中，如果某台主機確認為廣告主機後，你可以透過下面的論壇連結回報給我們。 hosts-file.net的 Hosts Inbox。

已存於快取中的廣告

某些軟體在下載時，會同時將廣告儲存在快取檔中，因此在這些軟體仍然會有廣告出現。你可以透過 Android 應用程式清單中的刪除快取檔案來解決此問題。

開啟廣告阻擋後，某些應用程式無法執行了！

有些應用程式需要連線到被 AdAway 阻擋的主機名稱，或廣告主機被阻擋導致程式拒絕執行。請參考 https://github.com/AdAway/AdAway/wiki/ProblematicApps 中記載有問題的應用程式清單。或是找出需要連線的主機名稱後，將它加入到你的名單中的白名單，並回報至 AdAway BUG追蹤頁面。

無法在 Android 4.4+ 中執行

請在偏好設定中把hosts 檔案來源於 /data/data/hosts 變更為/data/hosts 或 /system/etc/hosts 並且重新執行廣告阻擋

退出功能在瀏覽網頁時是無效的。

可在 AdAway 的設定中啟用本地網頁伺服器作為解決辦法。
