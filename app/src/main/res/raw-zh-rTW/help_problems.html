在 Android 9+ 上複製 hosts 檔案失敗

新版的 Android 使用唯讀的 /system 分割區
。 如果您使用 Magisk 作為 root 解決方案，請確保啟用無系統 hosts 模組，然後重新啟動手機。

分割區空間不足

嘗試在偏好設定中將目標 hosts 檔案更改為 /data/data/hosts（或 /data/hosts），然後再次應用 AdAway。

Chrome 仍然顯示廣告

請在 Chrome 設定中停用lite 模式（以前稱為data saver）。此功能使 Chrome 可以使用私有 DNS 解決方案，從而繞過 AdAway。

某個應用程式中未阻擋廣告！

某些主機名稱可能在「主機源」所提供的 hosts 來源中缺失，或者是應用程式將廣告圖片直接嵌入，因此不需要網路存取。

您可以透過 AdAway 的記錄 DNS 請求（選單 -> 記錄 DNS 請求）功能，找出還需要被阻擋的額外主機名稱。

透過長按記錄中的項目，將可疑的主機名稱加入您自己的「黑名單」，並在驗證後報告到我們的 GitHub 問題追蹤系統。

已存於快取中的廣告

有時，應用程式會在下載後快取廣告，導致某些應用程式內仍然出現廣告。您可以嘗試在 Android 的應用程式清單中刪除這些應用程式的快取，以解決此問題。

某個應用程式失效了！

有些應用程式需要與 AdAway 所阻擋的某個主機名稱進行通訊，或者在應該提供廣告的主機名稱無法連接時會拒絕運行。請前往 https://github.com/AdAway/AdAway/wiki/ProblematicApps 查看已知有問題的應用程式清單。或者，找出需要的主機名稱並將其加入您的白名單，然後報告到 AdAway 的問題追蹤系統。

無法在 Android 4.4+ 中執行

嘗試在偏好設定中更改目標 hosts 檔案的路徑，從 /data/data/hosts 改為 /data/hosts 或 /system/etc/hosts，然後重新套用 AdAway。

退出功能在瀏覽網頁時是無效的。

可在 AdAway 的設定中啟用本地網頁伺服器作為解決辦法。
